Tutorial Ubuntu Server Paket 1

Spesifikasi Server:
IP Server: 192.168.30.10
OS: Ubuntu Server 24.04
Akses: VirtualBox / SSH

1. Penginstalan SSH

SSH digunakan agar kita bisa mengontrol server dari Windows tanpa harus membuka VirtualBox.

Install SSH

sudo apt update
sudo apt install ssh -y

Cek Status SSH

systemctl status ssh

Pastikan muncul:

active (running)

Login dari Windows

Buka Command Prompt (CMD) di Windows, lalu ketik:

ssh username@192.168.30.10
Jika muncul pertanyaan password, masukkan password Ubuntu Server, bukan password database.

2. DNS Server (BIND9)

DNS berfungsi menerjemahkan nama domain menjadi IP. Di sini kita akan menggunakan domain lokal: 

lab-smk.xyz

Install BIND9

sudo apt install bind9 -y

Konfigurasi Zone

sudo nano /etc/bind/named.conf.local
Tambahkan: 
zone "lab-smk.xyz" {
    type master;
    file "/etc/bind/db.lab-smk";
};

Buat File Zone

sudo cp /etc/bind/db.local /etc/bind/db.lab-smk
sudo nano /etc/bind/db.lab-smk
Isi: 
$TTL 300
@   IN  SOA lab-smk.xyz. root.lab-smk.xyz. (
        2026012201
        3600
        900
        604800
        300 )

@       IN  NS  ns.lab-smk.xyz.
@       IN  A   192.168.30.10
ns      IN  A   192.168.30.10
www     IN  A   192.168.30.10
monitor IN  A   192.168.30.10

Restart DNS

sudo systemctl restart bind9

Pengetesan (Windows)

Atur DNS Windows ke 192.168.30.10, lalu:

ping lab-smk.xyz
ping www.lab-smk.xyz
ping monitor.lab-smk.xyz

4. Zabbix Server (Versi Aman & Pasti Jalan)

Bagian ini memakai cara manual schema database karena pada beberapa sistem folder sql bawaan Zabbix tidak tersedia.

Update Sistem

sudo apt update && sudo apt upgrade -y

Install MariaDB

sudo apt install mariadb-server -y

Secure MariaDB

sudo mysql_secure_installation
Gunakan jawaban: 
Switch to unix_socket authentication ‚Üí Y
Set root password ‚Üí N
Remove anonymous users ‚Üí Y
Disallow root login remotely ‚Üí Y
Remove test database ‚Üí Y
Reload privilege tables ‚Üí Y

Buat Database Zabbix

sudo mysql
CREATE DATABASE zabbix CHARACTER SET utf8mb4 COLLATE utf8mb4_bin;
CREATE USER 'zabbix'@'localhost' IDENTIFIED BY '1';
GRANT ALL PRIVILEGES ON zabbix.* TO 'zabbix'@'localhost';
FLUSH PRIVILEGES;
EXIT;
Password database adalah 1. Jangan memasukkan password Ubuntu saat diminta password MySQL.

Install Repository Zabbix

wget https://repo.zabbix.com/zabbix/7.0/ubuntu/pool/main/z/zabbix-release/zabbix-release_latest_7.0+ubuntu24.04_all.deb
sudo dpkg -i zabbix-release_latest_7.0+ubuntu24.04_all.deb
sudo apt update

Install Zabbix

sudo apt install zabbix-server-mysql zabbix-frontend-php zabbix-apache-conf zabbix-agent -y

Download Schema Manual

cd ~
wget https://cdn.zabbix.com/zabbix/sources/stable/7.0/zabbix-7.0.0.tar.gz
tar -xzf zabbix-7.0.0.tar.gz
cd zabbix-7.0.0/database/mysql

Import Database (URUTAN WAJIB)

mysql -u zabbix -p zabbix < schema.sql
mysql -u zabbix -p zabbix < images.sql
mysql -u zabbix -p zabbix < data.sql
Password: 
1

Konfigurasi Zabbix

sudo nano /etc/zabbix/zabbix_server.conf
Pastikan: 
DBName=zabbix
DBUser=zabbix
DBPassword=1

Restart Service

sudo systemctl restart zabbix-server apache2 zabbix-agent
sudo systemctl enable zabbix-server apache2 zabbix-agent

Pada bagian ini kita akan mengatur Firewall Filter Rules di MikroTik untuk mengamankan jaringan antar VLAN serta akses ke router.

Masuk ke MikroTik melalui:
IP ‚Üí Firewall ‚Üí Filter Rules

aÔ∏è‚É£ Blok Akses VLAN 20 ke VLAN 10

Aturan ini digunakan untuk melarang VLAN 20 mengakses jaringan VLAN 10.

Catatan penting:
Rule ini harus berada DI ATAS rule allow agar benar-benar bekerja. 

Chain        : forward
Src. Address : 192.168.20.0/24
Dst. Address : 192.168.10.0/24
Action       : drop
Comment      : BLOCK VLAN20 to VLAN10

bÔ∏è‚É£ Izinkan VLAN 30 ke VLAN 10

Berbeda dengan VLAN 20, VLAN 30 diizinkan untuk mengakses VLAN 10. Biasanya VLAN ini digunakan untuk admin atau monitoring. 

Chain        : forward
Src. Address : 192.168.30.0/24
Dst. Address : 192.168.10.0/24
Action       : accept
Comment      : ALLOW VLAN30 to VLAN10
Pastikan rule ini berada DI ATAS rule DROP VLAN20, karena MikroTik membaca firewall dari atas ke bawah.

cÔ∏è‚É£ Pengamanan Brute-Force Login (SSH & Winbox)

Untuk mencegah serangan brute-force (login berulang-ulang), kita gunakan Address List.

Rule 1 ‚Äî Deteksi Brute-Force

Chain              : input
Protocol           : tcp
Dst Port           : 22,8291
Connection State   : new
Action             : add-src-to-address-list
Address List       : brute-force
Address List Time  : 1d

Rule 2 ‚Äî Blok Brute-Force

Chain            : input
Src Address List : brute-force
Action           : drop
Comment          : BLOCK BRUTE FORCE
Artinya: jika ada IP mencoba login berulang kali, IP tersebut akan otomatis diblokir selama 1 hari.

dÔ∏è‚É£ Logging Aktivitas Penting Router

Rule ini digunakan untuk mencatat aktivitas akses ke router agar admin dapat melakukan monitoring. 

Chain      : input
Action     : log
Log Prefix : ROUTER-ACCESS
Jangan menambahkan terlalu banyak rule log karena dapat membuat router menjadi berat.

eÔ∏è‚É£ Pembatasan Akses Router (SSH & Winbox)

Akses ke router hanya diperbolehkan dari VLAN tertentu, yaitu VLAN 10 dan VLAN 30.

Allow VLAN 10

Chain        : input
Src Address : 192.168.10.0/24
Protocol    : tcp
Dst Port    : 22,8291
Action      : accept

Allow VLAN 30

Chain        : input
Src Address : 192.168.30.0/24
Protocol    : tcp
Dst Port    : 22,8291
Action      : accept

Drop Akses Selain Itu

Chain      : input
Protocol   : tcp
Dst Port   : 22,8291
Action     : drop
Comment    : DROP OTHER ACCESS
Dengan aturan ini, hanya perangkat dari VLAN 10 dan VLAN 30 yang bisa mengakses router menggunakan SSH dan Winbox.
